1. Upload the webpage

先請你上到Github，登入以後，請點選右上角的New repository(有點像是創立一個虛擬的主機放我的網頁)，Repository name:可以取任何的名字，也可以跟本地端一樣的名字，接下來就是建立這個repository，

當然，現在這個虛擬的主機什麼東西都沒有，接下來這個步驟稍微複雜，請多看幾遍，

a. 現在你的網頁寫好(假設放在本地端的資料夾FirstWebsite)
b. 我的習慣是再創一個新的資料夾，把FirstWebsite的所有檔案複製進去(這個資料夾叫做FirstWebsite_Github)
c. 接下來請打開你的cmder或是其他shell，
d. 用cd 把工作的資料夾換到FirstWebsite_Github，
e. 開始上傳作業:
   依序輸入
   1. git init：初始化本地的資料夾
   2. git add -A: 把所有檔案加進去
   3. git commit -m "提交說明": 把檔案準備推送到GitHub
   ex: git commit -m "第一次上傳"
   4. git remote add origin "你的GitHub repository地址" (下面解釋)
   到你的repository裡面，點選右邊的clone or download，把連結的地址貼到上面 
   ex:git remote add origin "https://github.com/ironmancool/PoChunChenWorld.git" 
   5. git pull origin master: 把本地資料夾的變化連接到你在GitHub上創立的repository的主分支
   6. git push -u origin master: 將本地的資料夾的程式碼，推送到GitHub上
   7. git status: 確認有沒有成功的訊息。

完成之後，請到你的GitHub repository上確認，所有檔案都有真的上傳上去，如果成功的話，應該會出現一堆檔案+"第一次上傳"+時間，

恭喜你，如果都沒有問題的話，就可以做最後一步了，
請到你的repository的settings裡，
區塊GitHub Pages中，source的部分，請點選master branch，接下來幾秒後會跑出一個連結，剛點進去可能會一直404錯誤，不過別擔心(只要你的index.html有真的上傳)，過幾秒就會出現網頁內容了。
